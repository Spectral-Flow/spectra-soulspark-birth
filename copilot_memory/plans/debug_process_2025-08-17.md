# Debug Process for Copilot Memory Assistant

1. **Context Gathering**: Collect project structure, main features, known issues, and code organization.
2. **Memory Bank Update**: Log new findings, fixes, and plans in the Copilot memory bank (JSON/Markdown).
3. **Error Detection**: For any code snippet, check for errors, improvements, or optimizations.
4. **Suggestion Mode**: Reference past solutions and snippets from the memory bank when suggesting fixes.
5. **Iterative Logging**: After each debug/fix, log the solution and reasoning in the memory bank.
6. **Retrieval**: When a new issue arises, search the memory bank for relevant past solutions and guidance.
7. **Annotation**: Allow edits/annotations to memory entries for clarity and future reference.

---

This process ensures persistent, organized, and evolving debugging support for the developer.
